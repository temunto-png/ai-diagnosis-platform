## Step 1: 現状分析

### Aさん（SEO・流入）の分析

**強み**
- AI診断という独自機能が「写真で診断」「AIカビ判定」等の新興キーワードで差別化できる
- Astro SSG + Cloudflare Workers の構成は Core Web Vitals で圧倒的に有利（LCP・FID・CLS全て良好になりやすい）
- 構造化マークアップ（SchemaOrg）・OGP・robots.txt・sitemap が既に実装済み
- ガイド記事がH2/H3構造・手順形式で、Google のHowTo リッチリザルト獲得の素地がある

**弱み**
- 記事数10本は絶対数として少なく、トピッククラスター戦略が未成熟
- 「賃貸 原状回復」「DIY 補修」は大手メディア（SUUMO・LIFULL等）やリフォーム業者が上位を占め、ドメインパワーで劣る
- 車関連記事（タイヤ・車傷）と住居関連記事（壁紙・カビ・床）が混在し、サイトのトピック権威性が分散している
- 内部リンク構造が不明確 — 記事間のリンクやピラーページが不足している可能性

**競合ギャップ分析**
| キーワード群 | 競合強度 | 撮偵の勝ち筋 |
|-------------|---------|-------------|
| 「壁紙 補修 DIY」「クロス 剥がれ 直し方」 | 中 | 手順記事＋AI診断CTA で差別化可能 |
| 「カビ 除去 方法」「黒カビ 取り方」 | 高 | 「カビ AI判定」で独自角度を狙える |
| 「賃貸 退去 原状回復 費用」 | 非常に高 | 正面勝負は困難。「賃貸 退去前 セルフチェック AI」等のニッチ狙い |
| 「フローリング 傷 補修 100均」 | 中 | 商品提案付き診断は競合にない |
| 「タイヤ 交換時期 判断」 | 高 | DIY住居と離れているため優先度下げ |

**AI診断のSEO活用ポテンシャル**
- 「AIでカビ診断」「写真で壁紙の傷を判定」等のキーワードは検索ボリュームこそ少ないが、**競合ゼロに近い**
- Programmatic SEO の観点で、診断結果ページをインデックス可能にすれば「○○ 補修方法」の長尾キーワードを自動的にカバーできる
- E-E-A-T の「Experience（体験）」軸で、ユーザーが実際にAIを使って診断→補修した体験をUGC的に蓄積できれば強力

**収益構造の設計方針**
- **短期**: 物販アフィリエイト（Amazon＋楽天）が確実。診断結果→道具・材料提案→購入リンクの導線が自然
- **中期**: A8.net のリフォーム見積もり案件（1件 3,000〜10,000円）は「DIYでは難しい」と診断した場合の出口として最適

- 注意: 審査中のAmazonアソシエイトは3件の適格な売上が必要 → 記事側のリンクを最適化して早期クリアを狙う

## Step 4: TOP5 実行計画

### 施策1: Amazon アソシエイト審査クリア最適化

**概要**
Amazon アソシエイトの審査は180日以内に3件の適格売上が条件。現在審査中のため、記事内リンクのクリック率を最大化し、早期に3件をクリアする。審査落ちは再申請が面倒なため、最優先で対処する。

**手順**
1. 各ガイド記事のAmazonリンクの配置を確認し、「手順の直後」「まとめセクション」の2箇所にリンクを集約
2. リンクテキストを「〇〇を見る →」から「この補修に必要な〇〇をAmazonで確認（税込〇〇円〜）」に変更し、クリック意図を明確化
3. 最もPVが多い記事（壁紙補修・カビ除去あたり）のリンクを重点的に最適化
4. 診断結果ページにもAmazonリンクが正しく表示されていることを確認
5. GA4でリンククリックイベントを追跡設定し、効果測定の基盤を作る

**KPI（成功指標）**
- 1週間以内にAmazonリンクのCTR 3%以上
- 2週間以内に3件の適格売上達成 → 審査通過

**注意点・落とし穴**
- Amazon アソシエイトのリンクは `rel="sponsored"` を付けること（SEOペナルティ回避）
- 価格表示は「〇〇円〜」とし、正確な価格は記載しない（Amazon規約で価格のキャッシュ表示は禁止）
- 過度なリンク設置はユーザー体験を損ない、逆にCVRが下がる。1記事あたり3〜5リンクが適正

### 施策2: DiagnosisCTA コンポーネント実装（L3）

**概要**
記事上部・末尾にカード型バナーで「AIで無料診断する」CTAを表示し、記事→診断ツールへのコンバージョン率を向上させる。設計承認済み（L3タスク）。

**手順**
1. `src/components/DiagnosisCTA.astro` を作成 — カード型デザイン、診断ツール種別（DIY補修/カビ）をpropsで受け取り
2. 記事テンプレート `src/pages/guide/[slug].astro` に上部（タイトル直下）と末尾（まとめセクション後）の2箇所に挿入
3. 記事のfrontmatterに `diagnosisType: "diy-repair" | "kabi-diagnosis"` フィールドを追加し、記事内容に合った診断ツールに誘導
4. CTAのテキストに「累計X人が診断済み」（L4実装と連動）を含めて社会的証明を付与
5. GA4でCTAクリックイベントを計測

**KPI（成功指標）**
- 記事→診断ツール遷移率 15%以上
- 診断ツール利用数が週20%増加

**注意点・落とし穴**
- 上部CTAは記事本文の邪魔にならない控えめなデザインにする（ユーザーが「広告だ」と感じると離脱する）
- モバイルでの表示崩れに注意（ターゲット層はスマホ利用が中心）

### 施策3: Google Search Console 登録＆初期分析

**概要**
GSC未登録の場合、検索パフォーマンスが完全にブラインド状態。インデックス状況・検索クエリ・CTRを把握し、データに基づくSEO改善の基盤を即座に構築する。

**手順**
1. Google Search Console にサイト（satsu-tei.com）を登録（DNS認証 or HTMLタグ認証）
2. sitemap.xml を手動送信してインデックスを促進
3. 全10記事のインデックス状況を確認し、未インデックスがあれば個別にURL検査→インデックス登録リクエスト
4. 1週間後にパフォーマンスレポートを確認 — 実際に流入しているクエリとCTRを把握
5. 表示回数が多いがCTRが低いクエリを特定し、タイトル・ディスクリプション改善の優先リストを作成

**KPI（成功指標）**
- 全ガイド記事10本がインデックス済み
- 1ヶ月後に検索クエリデータ50件以上蓄積

**注意点・落とし穴**
- Cloudflare を使っている場合、DNS認証がスムーズ（TXTレコード追加）
- GSCのデータ反映には2〜3日かかるため、登録後すぐにデータは見られない
- AdSenseメタタグが既に設置済みとのことだが、GSCの認証タグとは別物なので混同しない

### 施策4: 既存記事の内部リンク最適化

**概要**
10記事が孤立状態だと、Googleのクローラーがサイト構造を理解できず、各記事の評価が分散する。関連記事への相互リンクを追加し、トピック間の関連性を明確化する。

**手順**
1. 10記事の内容を確認し、リンクマップを作成（どの記事からどの記事に自然にリンクできるか）
2. 各記事の本文中に2〜3個の自然な文脈リンクを追加（例: 壁紙補修記事→壁紙シール記事、カビ記事→カビ予防記事）
3. 各記事の末尾に「関連ガイド」セクションを追加し、2〜3件の関連記事カードを表示
4. 診断ツールページからも「関連ガイド記事」としてリンクを設置
5. リンク設置後、GSCでクロール状況をモニタリング

**KPI（成功指標）**
- 全記事が少なくとも3つ以上の内部リンクを受けている状態
- 直帰率が5〜10%低下
- ページ/セッションが1.5以上に向上

**注意点・落とし穴**
- 車関連記事（タイヤ・車傷）と住居関連記事を無理にリンクしない — 不自然なリンクは逆効果
- アンカーテキストはキーワードを含むが自然な文言にする（「こちら」「詳しくはこちら」は避ける）

### 施策5: SNSアカウント開設＆プロフィール最適化

**概要**
SNSファネルがゼロの状態は最大の機会損失。Instagram・Xのアカウントを即座に開設し、サイトへの流入経路を新たに確保する。

**手順**
1. Instagram ビジネスアカウントを作成 — ユーザー名: `@satsutei` or `@satsu_tei`
   - プロフィール: 「📷 撮るだけAI補修診断｜賃貸DIYの味方」
   - リンク: satsu-tei.com（Linktreeは不要、直リンクで十分）
2. X（Twitter）アカウントを作成 — ユーザー名: `@satsutei`
   - プロフィール: 「写真を撮るだけでAIが補修方法を診断。賃貸DIYの不安を解消します。」
   - 固定ツイート: サービス紹介 + 診断ツールリンク
3. 両アカウントのプロフィール画像・ヘッダー画像をサイトのビジュアルと統一
4. 初回投稿として「サービス紹介」投稿を公開（サイトのスクリーンショット + 概要）
5. コンテンツカレンダー（最低2週間分）を作成し、投稿を開始

**KPI（成功指標）**
- 1週間以内に両アカウント開設完了
- 初月でInstagram フォロワー100人、X フォロワー50人
- SNS経由のサイト流入 月間50セッション以上

**注意点・落とし穴**
- ユーザー名が取られている場合は `satsutei_diy` 等のバリエーションを用意しておく
- 初期はフォロワーゼロなので、ハッシュタグ戦略が重要（#賃貸DIY #原状回復 #カビ対策 等）
- 投稿頻度を高くしすぎて継続できなくなるより、週2〜3投稿で確実に続ける方が重要

## Step 5: 専門家チームの総合提言

**3ヶ月後の最重要目標**: 月間オーガニック流入500セッション以上、月間診断利用数200件以上、アフィリエイト初収益月1万円達成。この3指標が揃えば「PMF（プロダクト・マーケットフィット）の初期兆候あり」と判断できる。

**まず最初にやるべき一手**: Amazon アソシエイト審査の早期クリア。審査が通らなければマネタイズの柱が立たず、全ての施策がPVだけで終わる。今週中に記事内リンクを最適化し、DiagnosisCTA実装とGSC登録を同時並行で進めること。

**長期的な収益安定の核心戦略**: 「AI診断」という唯一無二のUXをコンテンツSEOとSNSバイラルの両方で訴求し、**診断利用→商品購入（物販アフィ）＋プロ依頼（CPA案件）の二層マネタイズ構造**を構築すること。AI診断の精度と信頼性がこの構造全体の基盤であり、それが競合優位の源泉でもある。
